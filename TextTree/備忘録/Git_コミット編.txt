<h3>コミットってなに？</h3>
<div class="text_base">
ファイルやディレクトリの追加・変更を、リポジトリに記録することを「コミット」というらしい。。。

ふむ。
要するにだ！現在の状態を記録することなんじゃな！
</div>
&nbsp;

<h3>コミットの前に状態を確認</h3>
<div class="text_base">
記録を残す前に必ず、現在の状態を確認すべし。

＞＞ git status

これをつかえば、状態を確認できるんじゃ！
</div>
&nbsp;

<h3>コミットしてみた</h3>
<div class="text_base">
いよいよコミットじゃな！

＞＞　git commit -a -m "コメントの記入"

ん？「-a」とはなんぞや？
これは、変更したファイルを自動で検出してくれるオプションらしい。。。まあ「all」　の　「a」　で全検索的な感じじゃな！

むむ？「-m　"コメント記入"」ともあるぞい？
これは・・・
一度試しに、
＞＞　git commit -a 

でしてみると一発でわかるそうなんじゃが・・・

あ！エディタが起動した！
そうなんじゃな。エディタの起動をさせずに記録のコメントを記入できるんじゃ！

基本的には、
＞＞　git commit -a -m "コメントの記入"
で間違いないらしいぞ！

</div>

<h3>補足：ファイルを追加したとき</h3>
<div class="text_base">
注意すべきは、ファイルを追加したときじゃ。
コミットができんぞ！
先に、ファイルの追加を指定するんじゃ！

＞＞　git add ＜追加したファイル＞

ちなみに、
＞＞　git add .
＞＞　git add -a
上の二つのどちらかをすると、「すべての」追加したファイルを指定できるんじゃ！
